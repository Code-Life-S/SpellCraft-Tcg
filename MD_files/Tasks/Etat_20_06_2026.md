# Etat du jeu - 20 Juin 2026

## Architecture generale

- **Framework** : Vanilla JS, HTML5, CSS3 (aucune librairie externe)
- **Cycle de vie** : `initialize()` -> `show()` -> `hide()` -> `destroy()` via `BaseScreen`
- **Stockage** : localStorage uniquement
- **Rendu** : Manipulation DOM directe
- **Templates HTML** : Fichiers .html separes avec fallback inline
- **Ciblage** : Navigateur web desktop/mobile (fichiers statiques, pas de serveur)

## Systeme de reactions elementaires

Implemente dans `game/elementalReactionsConfig.js` et `game/elementalReactionsManager.js` :

- **Gele** (Froid) -- duree 2 tours ennemi, saute l'attaque, persiste jusqu'au tour joueur suivant
- **Brulure** (Feu) -- duree 3 tours, 1 degat au debut du tour ennemi (2 tics de degats)
- **Electrise** (Foudre) -- duree 2 tours, persiste jusqu'au tour joueur suivant
- **Melt** (Feu sur Gele) -- degats x2, consume Gele
- **Overload** (Feu sur Electrise) -- +2 degats AOE a tous, consume Electrise
- **Shatter** (Foudre sur Gele) -- +3 degats bonus, consume Gele
- Fonds d'ecran animes par statut (CSS gradients avec animations)
- Icones de statut superposees sur les cartes ennemies

## Modes de jeu

### Mode Vague Defense
- Selection de deck, mulligan, mana 1-10 croissant
- 3-4 ennemis generes aleatoirement (stats fixes)
- **1 seule vague** -- victoire = tous les ennemis morts
- Stats localStorage : `bestWave`, `totalVictories`, `spellsCast`

### Mode Arena
- Phase 1 : Builder -- draft 10 cartes (choix 1 parmi 3, 10 fois)
- Phase 2 : Adventure -- 12 rounds, ennemis progressifs (HP/attaque + taille de main)
- Soins + ajout carte + upgrade entre rounds
- Points de vie max augmentent aux rounds 3/6/9
- Defaite = fin de run
- Persistance via localStorage (`arenaBuilderState`)

### Deck Builder
- Creer/editer decks de 30 cartes (max 2 copies par carte)
- Deck par defaut : Starter, Aggressive Spellcaster, Control Mage
- Filtres par nom, cout mana, rarete

## Ennemis

**12 types generiques** (identiques mecaniquement) dans chaque mode :

Goblin, Orc, Skeleton, Wolf, Bandit, Spider, Dark Mage, Minotaur, Wraith, Gargoyle, Demon, Vampire

**Comportements** :
- Attaque de base (degats physique)
- Attaquent un par un en sequence (1.5s interval en vague, 0.35s en arena)
- Peuvent etre geles/brules/electrises via reactions elementaires
- 4 animations de mort differentes en mode vague

**Aucune capacite speciale** : pas de soin, bouclier, invocation, AOE, resistances, boss

---

## Systemes de progression

**Tres minimal** -- pas d'XP, niveau, deblocage, ou collection de cartes.

Ce qui existe en localStorage :
- `bestWave`, `totalVictories`, `spellsCast` (stats vague)
- `soundEnabled`, `musicEnabled` (audio)
- `historyVisible`, `deckVisible` (UI)
- `spellcaster_decks` + `spellcaster_decks_version` (decks)
- `arenaBuilderState` (etat arena)

Aucun systeme de :
- XP / niveaux joueur
- Succes / achievements
- Deblocage de cartes
- Monnaie / boutique
- Boss / ennemis speciaux
- Defis quotidiens

## Audio et visuel

**Son** (`audio/soundManager.js`) : synthetise via Web Audio API
- Sons d'interface, sorts (par element), combat, victoire/defaite
- Musique de fond synthetisee (inspiration Zelda)
- Controles volume SFX/musique separes
- **Aucun fichier audio** (pas de .mp3/.wav)

**Effets visuels** :
- Nombres flottants (degats, soin, bouclier)
- Impacts de sort colores par element
- Trainees de particules
- Particules AOE
- Secousse d'ecran
- Lueur de soin/bouclier/mana
- Animations de mort (4 variantes)
- Overlays de statut (gele/brulure/electrise)
- Transitions d'ecran (fade + slide)
