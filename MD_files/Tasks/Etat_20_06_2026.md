# Etat du jeu - 20 Juin 2026

## Architecture generale

- **Framework** : Vanilla JS, HTML5, CSS3 (aucune librairie externe)
- **Cycle de vie** : `initialize()` -> `show()` -> `hide()` -> `destroy()` via `BaseScreen`
- **Stockage** : localStorage uniquement
- **Rendu** : Manipulation DOM directe
- **Templates HTML** : Fichiers .html separes avec fallback inline
- **Ciblage** : Navigateur web desktop/mobile (fichiers statiques, pas de serveur)

## Systeme de classes (3 classes)

Defini dans `game/classConfig.js` et `game/classManager.js` :

| Classe | ID | Passif |
|--------|----|--------|
| Pyromancien | pyromancer | +1 degat aux sorts de feu |
| Cryomancien | cryomancer | Ennemis geles au debut du combat |
| Necromancien | necromancer | +1 PV max +1 PV courants a chaque mort ennemie |

**Mecanismes** :
- `ClassManager` est un singleton avec `_activeClassId` + localStorage
- `ClassManager.getFireDamageBonus()` : retourne +1 si classe active = pyromancer
- `ClassManager.onEnemyDeath()` : si necromancer, retourne `{health, maxHealth}` avec +1 chaque
- `setActiveClass()` / `getActiveClassId()` pour persistance
- `getAvailableClasses()` / `getRandomClasses(count)` pour selection aleatoire

## Systeme de reactions elementaires

Implemente dans `game/elementalReactionsConfig.js` et `game/elementalReactionsManager.js` :

- **Gele** (Froid) -- duree 2 tours ennemi, saute l'attaque, persiste jusqu'au tour joueur suivant
- **Brulure** (Feu) -- duree 3 tours, 1 degat au debut du tour ennemi (2 tics de degats)
- **Electrise** (Foudre) -- duree 2 tours, persiste jusqu'au tour joueur suivant
- **Melt** (Feu sur Gele) -- degats x2, consume Gele
- **Overload** (Feu sur Electrise) -- +2 degats AOE a tous, consume Electrise
- **Shatter** (Foudre sur Gele) -- +3 degats bonus, consume Gele
- Fonds d'ecran animes par statut (CSS gradients avec animations)
- Icônes de statut superposees sur les cartes ennemies

**Correction appliquee** : `applySpellEffect()` passe desormais `card.element` (et non `spellType`) a `applyDamageWithElement()`, ce qui permet aux cartes de classe (Conflagration=fire, Blizzard=frost) de declencher correctement les reactions et statuts.

## Modes de jeu

### Mode Vague Defense
- Selection de deck, mulligan, mana 1-10 croissant
- 3-4 ennemis generes aleatoirement (stats fixes)
- **1 seule vague** -- victoire = tous les ennemis morts
- Stats localStorage : `bestWave`, `totalVictories`, `spellsCast`

### Mode Arena
- Phase 1 : Class Select (3 classes aleatoires) -> Builder -- draft 10 cartes (choix 1 parmi 3, neutre + classe seulement)
- Phase 2 : Adventure -- 12 rounds, ennemis progressifs (HP/attaque + taille de main)
- Soins + ajout carte (neutre ou classe courante) + upgrade entre rounds
- Points de vie max augmentent aux rounds 3/6/9
- Boss au round 12 (Roi Squelette / Mage Noir / Dragon)
- Capacites ennemies actives : Provocation, Enrage, Soigneur, Invocateur
- Defaite = fin de run
- Persistance via localStorage (`arenaBuilderState`)

### Deck Builder
- Creer/editer decks de 30 cartes (max 2 copies par carte)
- Deck par defaut : Starter, Aggressive Spellcaster, Control Mage
- Filtres par nom, cout mana, rarete, classe
- Choix de classe lors de la creation d'un nouveau deck

## Ennemis

**12 types generiques** (identiques mecaniquement) dans chaque mode :

Goblin, Orc, Skeleton, Wolf, Bandit, Spider, Dark Mage, Minotaur, Wraith, Gargoyle, Demon, Vampire

**Comportements** :
- Attaque de base (degats physique)
- Attaquent un par un en sequence (1.5s interval en vague, 0.35s en arena)
- Peuvent etre geles/brules/electrises via reactions elementaires
- 4 animations de mort differentes en mode vague

**Capacites ennemies** (arene rounds 4+, classique 50% chance) :
- Provocation : seul cet ennemi peut etre cible tant qu'il est vivant
- Enrage : +2 ATK permanent quand HP < 50%
- Soigneur : soigne l'allié le plus bas pour 2 a la fin du tour
- Invocateur : invoque un sbire 1/1 a la fin du tour

**Boss** (arene round 12 uniquement) :
- Roi Squelette : 20 HP, 4 ATK, invoque 2 sbires au debut de chaque tour
- Mage Noir : 15 HP, 3 ATK, 3 bouclier/tour, drain 2 PV joueur + auto-soin, etourdissement si bouclier brise
- Dragon : 30 HP, 5 ATK, souffle 2 degats AOE au joueur chaque tour, Enrage a <50% HP

## Systemes de progression

**Implementation complete** -- XP, niveaux, progression, deblocage de cartes et classes.

Fichier : `game/playerProgressionManager.js`

### XP et niveaux
- XP gagnee : 10 par kill, 25 par round/vague, 50 victoire / 20 defaite
- Palier par niveau : `niveau * 100` XP
- Boucle de level-up securisee (max 50 iterations)
- Stockage dans localStorage : `spellcaster_progression`

### Deblocage de cartes (neutres)
- 10 cartes de depart au niveau 1
- 6 cartes a debloquer reparties niveaux 2-7 :
  - Niv 2 : Mystical Insight
  - Niv 3 : Greater Heal
  - Niv 4 : Thunder Storm
  - Niv 5 : Arcane Missiles
  - Niv 6 : Meteor
  - Niv 7 : Divine Wrath
- Cartes verrouillees affichees grisees dans le deck builder avec indication du niveau requis
- Choix arena filtres pour n'inclure que les cartes debloquees

### Deblocage de classes
- Pyromancer : debloque par defaut
- Cryomancer : gagner un run arena avec Pyromancer
- Necromancer : gagner un run arena avec Cryomancer
- Classes verrouillees affichees avec cadenas + condition dans l'ecran de selection

### Affichage progression
- Barre XP et niveau dans le menu principal
- Indication de la prochaine carte ou classe a debloquer
